# 车衣车膜质保小程序 · 设计规格

**日期：** 2026-05-25  
**技术栈：** uni-app (Vue3 + TypeScript) + 微信云开发（CloudBase）  
**UI 库：** uview-plus  
**图表库：** uCharts  

---

## 一、项目概述

面向汽车改装/贴膜门店的微信小程序，提供两大核心功能：

- **管理员端**：登录后录入、编辑、删除车衣车膜质保记录，查看数据看板
- **车主端**：免登录，凭手机号 / 质保卡号 / 车牌号 / 车架号查询自己的质保信息

## 三、数据模型

### 集合 `records`（质保记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 云数据库自动生成 |
| `warranty_no` | string | 质保卡号，唯一，格式 `BM` + `YYYYMMDD` + 4位序号 |
| `owner_name` | string | 车主姓名 |
| `owner_phone` | string | 车主手机号 |
| `license_plate` | string | 车牌号 |
| `vin` | string | 车架号 |
| `car_model` | string | 车型 |
| `car_color` | string | 车辆颜色 |
| `product_brand` | string | 产品品牌 |
| `product_series` | string | 产品系列 |
| `product_model` | string | 产品型号 |
| `construction_parts` | string[] | 施工部位（如 `["全车", "引擎盖"]`） |
| `construction_date` | date | 施工日期 |
| `delivery_date` | date | 交车时间 |
| `warranty_years` | number | 质保年限（年） |
| `warranty_expire` | date | 质保到期日（系统自动计算：施工日期 + 质保年限） |
| `technician` | string | 施工师傅 |
| `total_price` | number | 施工总价（元） |
| `product_images` | string[] | 产品图片云存储 URL 数组（最多 9 张） |
| `construction_images` | string[] | 施工图片云存储 URL 数组（最多 9 张） |
| `created_by` | string | 录入管理员的 openid |
| `created_at` | date | 录入时间 |
| `updated_at` | date | 最后更新时间 |

**索引：** `warranty_no`（唯一）、`owner_phone`、`license_plate`、`vin`、`warranty_expire`

### 集合 `admins`（管理员白名单）

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `openid` | string | 微信 openid（唯一） |
| `name` | string | 管理员姓名 |
| `created_at` | date | 添加时间 |

## 四、页面结构

### 管理员端（登录后）

```
TabBar
├── 首页看板
│   ├── 本月施工量 / 总记录数统计卡片
│   ├── 施工趋势折线图（近 6 个月）
│   ├── 产品销量占比饼图
│   ├── 质保到期预警列表（30 天内到期）
│   └── 待回访客户提醒
│
├── 记录列表
│   ├── 搜索筛选（车牌 / 手机号 / 质保卡号 / 日期范围）
│   ├── 记录卡片列表（含质保状态标签）
│   └── 点击进入记录详情（可编辑 / 删除）
│
└── 我的
    ├── 管理员信息展示
    ├── 添加管理员（输入姓名 + 扫码获取 openid）
    └── 退出登录

独立页面
├── 新增记录（分步表单：基本信息 → 产品信息 → 图片上传）
└── 记录详情 / 编辑页
```

### 车主端（免登录）

```
查询首页（小程序默认首页）
├── 四种查询方式 Tab（手机号 / 质保卡号 / 车牌号 / 车架号）
├── 输入框 + 查询按钮
└── 查询结果页
    ├── 车辆基本信息
    ├── 产品 & 施工信息
    ├── 质保状态标签（有效 / 即将到期 / 已过期，颜色高亮）
    ├── 产品图片 & 施工图片（可放大预览）
    └── 无结果提示（引导联系门店）
```

**入口设计：** 小程序首页为车主查询页，底部固定"管理员登录"文字链接。

## 五、云函数设计

| 函数名 | 触发方 | 职责 |
|--------|--------|------|
| `login` | 管理员端 | 获取 openid，校验白名单，返回登录态 |
| `addRecord` | 管理员端 | 原子生成质保卡号，写入记录 |
| `updateRecord` | 管理员端 | 更新记录，同步 `updated_at` |
| `deleteRecord` | 管理员端 | 删除记录，同步清理云存储图片 |
| `queryRecord` | 车主端 | 按手机号 / 卡号 / 车牌 / 车架号查询，脱敏后返回 |
| `getDashboard` | 管理员端 | 聚合统计：施工趋势、产品占比、到期预警 |
| `addAdmin` | 管理员端 | 向 admins 集合写入新管理员 |

## 六、关键交互与错误处理

### 图片上传
- 产品图片、施工图片各最多 9 张
- 上传前本地压缩，单张限 500KB 以内
- 单张失败可独立重试，不影响其他图片
- 删除记录时云函数同步删除对应云存储文件

### 质保卡号生成
- 格式：`BM` + `YYYYMMDD` + 4位序号（如 `BM202605250001`）
- 由 `addRecord` 云函数原子操作生成，防止并发重复

### 车主查询
- 手机号匹配多条时返回列表（同一车主多次施工）
- 手机号脱敏：`138****8888`
- 无结果显示友好提示，引导联系门店

### 管理员登录
- 首次登录获取 openid，云函数校验白名单
- 不在白名单提示"无权限，请联系管理员"
- 登录态存 `wx.setStorage`，7 天内免重新授权

### 数据看板
- 统计通过 `getDashboard` 云函数聚合，不拉全量数据到前端
- 到期预警：`warranty_expire` 在今天至 30 天后之间，进入首页实时查询

## 八、非功能性要求

- **云存储配额**：按需开通，图片建议设置 CDN 加速
- **云数据库安全规则**：`records` 集合仅允许云函数读写，禁止客户端直接操作
- **小程序包体积**：分包加载，管理员端页面单独分包，主包仅保留车主查询
- **兼容性**：支持微信基础库 3.0.0+（2022年后主流机型均满足）
